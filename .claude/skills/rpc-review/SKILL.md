# RPC Review Skill

## Overview

実装後に、RPC 型安全性に関する違反パターンをチェックする。

---

## Check Instructions

### 型アサーション禁止
- `as SomeType` による型アサーションを RPC レスポンスに使っていないか
- 型推論できない場合は `InferResponseType` / `InferRequestType` で対応しているか

### API クライアントの使い分け
- `fetch` / `axios` を直接使っていないか（`rpc` クライアント経由か）
- 例外：リフレッシュトークンのエンドポイントは `lib/api-client.ts` の使用が許可されている

### バックエンドのパスエイリアス
- バックエンドファイルに `@/` パスエイリアスが追加されていないか（相対パスを使うこと）

---

## Procedure

1. 変更されたファイルを確認する
2. 各チェック項目を照合する
3. 以下の形式で報告する

---

## Output Format

違反がある場合：

```
## RPC Review 結果

### 違反あり
- **ファイル**: [ファイルパス:行番号]
- **違反内容**: 具体的な問題
- **修正方針**: 修正の方向性
```

違反がない場合：

```
## RPC Review 結果

チェック完了。問題なし。
```
